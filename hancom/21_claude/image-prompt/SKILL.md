이미지를 입력받아 화법, 기풍, 화면구도, 카메라 세팅 등을 분석하고 이미지 생성 AI에 바로 활용할 수 있는 프롬프트를 역산합니다.

## 처리 절차

1. **이미지 종류 판별**: 사진(포토그래피)인지 회화/일러스트인지 자동 판단합니다. 경계가 모호하면(예: 포토리얼 일러스트, AI 생성 이미지) 양쪽 관점을 모두 반영합니다.
2. **EXIF 메타데이터 확인**: 사진으로 판별되고 원본 파일에 EXIF가 남아있다면 조리개/셔터스피드/ISO/렌즈 정보를 먼저 확인합니다. 없으면 시각적 단서(피사계심도, 모션블러, 색온도 등)로 추정합니다.
3. **헤드라인 요약**: 본문 최상단에 장르 + 핵심 무드 키워드를 한 줄로 제시합니다 (예: "수채화풍 애니메이션 일러스트 · 파스텔톤, 명랑한 여름 무드").
4. **7개 고정 섹션으로 분석**: 아래 섹션 순서와 형식을 그대로 따릅니다.
5. **종합 프롬프트 조합**: 마지막에 전체 섹션의 영어 키워드를 하나로 묶은 프롬프트 문단을 추가합니다.

## 분석 섹션 (고정 7개, 이 순서 유지)

각 섹션은 다음 형식을 따릅니다: 한국어로 1~2문장의 근거 설명 + 영어 키워드 불릿 리스트(3~5개).

| # | 섹션 | 다루는 항목 |
|---|------|------------|
| 1 | 구도 (Composition) | 프레이밍, 피사체 배치, 화각, 여백, 시점(앵글) |
| 2 | 화법·매체 (Technique/Medium) | 그림: 붓터치·질감·매체 / 사진: 렌더링 스타일·후보정 느낌 |
| 3 | 기풍·무드 (Style/Mood) | 예술 사조/작가 유사성, 전반적 톤과 분위기 |
| 4 | 카메라 세팅 (Camera Settings) | 렌즈 화각, 조리개(피사계심도), 셔터스피드(모션블러 유무), 화이트밸런스/색온도. 값 앞에 `[실측값]`/`[추정값]` 표시. 회화·일러스트는 이 섹션 생략 |
| 5 | 색채·팔레트 (Color Palette) | 주조색, 보조색, 채도/명도 경향 |
| 6 | 조명 (Lighting) | 광원 방향/개수, 하드/소프트 라이트, 명암비 |
| 7 | 후처리·렌더링 느낌 (Post-processing) | 필름룩, 그레인, 비네팅, 색보정 스타일, (3D인 경우) 렌더 엔진 느낌 |

## 종합 프롬프트

7개 섹션의 영어 키워드를 자연어 서술형으로 통합해 하나의 프롬프트 문단으로 제공합니다. 특정 생성 도구의 문법(Midjourney `--ar` 등 파라미터 태그)에 종속되지 않는 범용 형태로 작성하고, 도구별 변환은 사용자가 요청할 때만 추가로 제공합니다.

## 다중 이미지 처리

- 기본: 이미지별로 독립된 리포트를 각각 출력합니다.
- 사용자가 "통합해서 분석해줘" 등으로 명시적으로 요청한 경우에만, 여러 이미지에 걸친 공통 스타일을 하나로 묶어 분석하고 통합 프롬프트를 제공합니다.

## 주의사항

- 카메라 세팅과 색채 분석은 본질적으로 추정이 섞입니다. 실측값이 아닌 모든 항목은 반드시 `[추정값]`으로 명시해 사용자가 신뢰도를 판단할 수 있게 합니다.
- 특정 작가/사조를 언급할 때는 "~풍", "~를 연상시키는" 등 유사성 표현을 사용하고 단정하지 않습니다.
